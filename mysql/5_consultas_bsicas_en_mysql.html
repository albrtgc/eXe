5. Consultas básicas en MySQL

En esta sección aprenderás a obtener y manipular información almacenada en una base de datos MySQL utilizando el lenguaje SQL (Structured Query Language). Trabajarás con las instrucciones más comunes de consulta (SELECT), inserción (INSERT), actualización (UPDATE) y eliminación (DELETE), así como con funciones de agrupación y filtrado.

Objetivos del bloque

  • Aprender a consultar datos mediante SELECT con condiciones, ordenaciones y alias.
  • Insertar, actualizar y eliminar registros utilizando instrucciones DML (Data Manipulation Language).
  • Aplicar funciones de agregación y agrupar resultados con GROUP BY y HAVING.
  • Usar subconsultas para resolver necesidades más complejas.

5.1 Instrucción SELECT: extracción de información

La sentencia SELECT permite consultar los datos almacenados en las tablas.

SELECT nombre, precio
FROM productos;

Para filtrar registros, utiliza WHERE:

SELECT nombre, precio
FROM productos
WHERE precio > 30;

Y para ordenar resultados:

SELECT nombre, precio
FROM productos
ORDER BY precio DESC;

Puedes cambiar el nombre mostrado de una columna usando AS:

SELECT nombre AS "Producto", precio AS "Precio (€)"
FROM productos;

Prueba rápida: Muestra todos los productos cuyo precio sea superior a 20 €, ordenados de mayor a menor precio.

5.2 Inserción, actualización y eliminación de datos

Estas operaciones se agrupan bajo el nombre de DML (Data Manipulation Language).

Inserción de registros

INSERT INTO productos (nombre, precio, stock)
VALUES ('Monitor 27"', 189.99, 5);

Actualización de registros

UPDATE productos
SET precio = precio * 0.9
WHERE precio > 100;

Eliminación de registros

DELETE FROM productos
WHERE stock = 0;

Recuerda: si no indicas una condición WHERE, el comando afectará a todos los registros de la tabla.

Ejercicio guiado: Inserta tres productos nuevos, reduce un 10 % el precio de los que cuesten más de 150 €, y elimina los que tengan stock cero.

5.3 Consultas con funciones de agregación

Las funciones de agregación permiten realizar cálculos sobre conjuntos de registros.

  • COUNT(): cuenta el número de registros.
  • SUM(): suma los valores de una columna.
  • AVG(): calcula la media.
  • MIN() y MAX(): devuelven el valor mínimo y máximo.
SELECT COUNT(*) AS total_productos,
       AVG(precio) AS precio_medio
FROM productos;

Uso de GROUP BY

Permite agrupar registros según el valor de una columna:

SELECT categoria, COUNT(*) AS total, AVG(precio) AS media
FROM productos
GROUP BY categoria;

Y puedes filtrar grupos con HAVING:

SELECT categoria, AVG(precio) AS media
FROM productos
GROUP BY categoria
HAVING AVG(precio) > 50;

Actividad práctica: Calcula el número total de productos, la media de precios y el stock medio por categoría.

5.4 Subconsultas y operadores

Una subconsulta es una consulta dentro de otra. Se utiliza para obtener datos que sirven como condición o valor en la consulta principal.

SELECT nombre, precio
FROM productos
WHERE precio > (
  SELECT AVG(precio)
  FROM productos
);

También puedes usar operadores como IN o BETWEEN:

SELECT nombre
FROM productos
WHERE categoria IN ('Informática', 'Accesorios')
  AND precio BETWEEN 50 AND 200;

Ejercicio propuesto: Muestra los productos cuyo precio sea superior a la media general y cuya categoría sea “Informática”.

5.5 Actividades finales

  1. Crea una tabla clientes con columnas: id, nombre, email, fecha_registro.
  2. Inserta al menos cinco registros.
  3. Realiza las siguientes consultas:
    • Clientes ordenados por fecha más reciente.
    • Clientes cuyo email contiene “gmail”.
    • Conteo total de clientes y promedio de antigüedad.

Autoevaluación

  • ¿Sé utilizar correctamente las cláusulas WHERE, ORDER BY y GROUP BY?
  • ¿Puedo insertar, modificar y eliminar registros de forma segura?
  • ¿Sé aplicar funciones de agregación y subconsultas simples?
