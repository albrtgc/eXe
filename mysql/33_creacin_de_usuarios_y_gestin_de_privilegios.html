3.3 Creación de usuarios y gestión de privilegios

En MySQL es recomendable no utilizar el usuario root para tareas diarias o conexiones desde aplicaciones. En su lugar, se deben crear usuarios específicos con privilegios limitados.

Creación de usuarios

CREATE USER 'appuser'@'%' IDENTIFIED BY 'secreta';

En este ejemplo, '%' permite el acceso desde cualquier host. Para mayor seguridad, puedes usar 'localhost' si el acceso será solo local.

Concesión de privilegios

GRANT ALL PRIVILEGES ON demodaw.* TO 'appuser'@'%';
FLUSH PRIVILEGES;

El primer comando otorga todos los privilegios sobre la base de datos demodaw y el segundo actualiza las tablas de permisos.

Comprobación de permisos

SHOW GRANTS FOR 'appuser'@'%';

MySQL devolverá una lista de los permisos activos para ese usuario.

Buenas prácticas

  • Usar contraseñas robustas y únicas para cada usuario.
  • Asignar privilegios mínimos necesarios para la tarea a realizar.
  • Evitar el uso de GRANT ALL PRIVILEGES si no es imprescindible.

Actividad práctica: Crea un usuario appuser con permisos sobre la base de datos demodaw. Comprueba los privilegios con SHOW GRANTS y documenta el resultado.
