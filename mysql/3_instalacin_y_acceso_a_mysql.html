3. Instalación y acceso a MySQL

En esta sección prepararás tu entorno de trabajo con MySQL y practicarás el acceso tanto por línea de comandos (CLI) como con clientes gráficos (GUI). Verás varias opciones de instalación para que elijas la que mejor encaje con tu equipo y necesidades (XAMPP, Docker o instalación nativa con MySQL Server + Workbench).

Objetivos del bloque

  • Instalar y/o levantar un servidor MySQL funcional en local.
  • Conectarse al servidor usando CLI (mysql) y GUI (phpMyAdmin o MySQL Workbench).
  • Realizar comprobaciones básicas: versión, bases de datos, usuarios y privilegios.
  • Aplicar buenas prácticas iniciales de seguridad (usuario de aplicación distinto de root).

Prerrequisitos y materiales

  • Sistema operativo Windows, Linux o macOS actualizado.
  • Permisos para instalar software o ejecutar contenedores Docker.
  • Conexión a internet para descargar los paquetes necesarios.

Opciones de instalación (elige una)

  1. XAMPP (rápido y sencillo). Incluye MySQL/MariaDB y phpMyAdmin en un panel único.
  2. Docker (reproducible y “limpio”). Levanta MySQL en un contenedor aislado.
  3. Instalación nativa (mayor control). MySQL Server + MySQL Workbench como cliente GUI.

Pasos resumidos de trabajo

  1. Instala/levanta MySQL con XAMPP, Docker o instalación nativa.
  2. Comprueba la conexión en CLI: mysql -u root -p y ejecuta SELECT VERSION();.
  3. Conéctate con GUI (phpMyAdmin o Workbench) para verificar host, puerto, usuario y contraseña.
  4. Crea un usuario de aplicación con permisos limitados sobre una base de datos concreta.

Ejemplo (Docker Compose)

version: "3.9"
services:
  db:
    image: mysql:8.0
    container_name: mysql_daw
    environment:
      - MYSQL_ROOT_PASSWORD=root
      - MYSQL_DATABASE=demodaw
    ports:
      - "3306:3306"
    volumes:
      - ./data:/var/lib/mysql
    restart: unless-stopped

Uso: en la carpeta del archivo, ejecuta docker compose up -d. Para parar: docker compose down.

Comandos básicos en CLI

# Conexión
mysql -u root -p

# Comprobaciones iniciales
SELECT VERSION();
SHOW DATABASES;

# Crear BD de pruebas
CREATE DATABASE demodaw CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
USE demodaw;

# Crear usuario de aplicación (ejemplo)
CREATE USER 'appuser'@'%' IDENTIFIED BY 'secreta';
GRANT ALL PRIVILEGES ON demodaw.* TO 'appuser'@'%';
FLUSH PRIVILEGES;

Acceso por GUI

  • phpMyAdmin (si usas XAMPP): abre http://localhost/phpmyadmin y autentícate.
  • MySQL Workbench: crea una nueva conexión indicando Host (localhost), Port (3306), User (root o appuser) y contraseña.

Buenas prácticas iniciales

  • No uses root para aplicaciones; crea un usuario específico con privilegios acotados.
  • Trabaja con utf8mb4 y una colación moderna (utf8mb4_general_ci o utf8mb4_0900_ai_ci si está disponible).
  • Guarda tus scripts SQL en un repositorio (carpeta o Git) para mantener trazabilidad.

Problemas comunes y soluciones

  • Puerto 3306 ocupado: cambia el puerto en la configuración o detén el servicio que lo usa.
  • Error de autenticación: verifica usuario/contraseña y host permitido ('usuario'@'localhost' vs 'usuario'@'%').
  • Contenedor no inicia (Docker): borra el volumen de datos si cambiaste versión mayor, revisa logs con docker logs mysql_daw.

Checklist rápido:

  • Servidor MySQL en ejecución ✔
  • Conexión por CLI y GUI verificada ✔
  • BD demodaw creada y en uso ✔
  • Usuario appuser con privilegios sobre demodaw ✔

Actividades sugeridas

  1. Instalación evidenciada: sube capturas de tu servidor en marcha y resultado de SELECT VERSION();.
  2. Usuario y permisos: crea appuser y demuestra que puede acceder a demodaw pero no a otras BD.
