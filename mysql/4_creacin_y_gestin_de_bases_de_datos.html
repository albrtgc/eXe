4. Creación y gestión de bases de datos

En esta sección comenzarás a diseñar y construir bases de datos en MySQL. Verás cómo crear esquemas y tablas, elegir tipos de datos adecuados, aplicar restricciones de integridad y estructurar la información siguiendo principios de normalización. El objetivo es que seas capaz de pasar de un modelo conceptual sencillo a una implementación relacional coherente y mantenible.

Objetivos del bloque

  • Crear bases de datos y tablas con CREATE DATABASE y CREATE TABLE.
  • Seleccionar tipos de datos apropiados y definir claves primarias.
  • Aplicar restricciones (NOT NULL, UNIQUE, CHECK, FOREIGN KEY).
  • Comprender y aplicar las tres primeras formas normales (1FN, 2FN y 3FN).
  • Desarrollar un pequeño proyecto guiado a modo de práctica.

Estructura de la sección

  1. 4.1 Fundamentos y creación de bases de datos — Conceptos clave, creación y modificación de tablas, inserción y consulta básica.
  2. 4.2 Normalización, restricciones y actividades prácticas — Integridad, formas normales, proyecto “Biblioteca” y actividades interactivas.

Requisitos previos

  • Tener un servidor MySQL operativo (instalado o en Docker).
  • Saber conectarse por CLI o GUI y ejecutar sentencias simples (SELECT VERSION()).

Metodología de trabajo

Partimos de ejemplos breves y guiados que podrás replicar en tu entorno. A continuación, aplicarás los conceptos en tareas prácticas con datos realistas. La sección culmina con un mini-proyecto que consolida el diseño relacional y el uso de restricciones.

Criterios de logro

  • La base de datos se crea con codificación utf8mb4 y colación adecuada.
  • Las tablas definen PK y tipos de datos correctos, sin redundancias evidentes.
  • Se aplican restricciones de integridad y se justifican las decisiones de diseño.
  • El modelo resultante cumple al menos 3FN para el caso de estudio planteado.

Actividad de activación: Piensa en una pequeña aplicación (p. ej., una librería o tienda de accesorios). Enumera tres entidades principales y, para cada una, define 4–5 atributos con su tipo de dato tentativo. ¿Qué tabla necesitaría una clave foránea?

Duración estimada: 2–3 horas · Tipo de contenido: Diseño e implementación relacional · Nivel: Inicial–Medio DAW
