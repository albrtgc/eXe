4.2 Normalización, restricciones y actividades prácticas

Una vez creada la estructura básica de una base de datos, el siguiente paso consiste en garantizar su integridad, coherencia y eficiencia. Para ello, se aplican reglas de diseño conocidas como formas normales y restricciones SQL.

Restricciones de integridad

  • PRIMARY KEY: garantiza la unicidad de cada registro.
  • FOREIGN KEY: establece relaciones entre tablas, preservando la coherencia referencial.
  • UNIQUE: evita duplicados en una columna.
  • CHECK: impone condiciones sobre los valores.
  • NOT NULL: impide valores nulos en campos obligatorios.

Ejemplo de restricciones

ALTER TABLE productos
  ADD CONSTRAINT uq_nombre UNIQUE (nombre),
  ADD CONSTRAINT chk_precio CHECK (precio >= 0);

Si intentas insertar un producto con el mismo nombre o con precio negativo, MySQL rechazará la operación.

Introducción a la normalización

La normalización consiste en dividir los datos en tablas lógicas que reduzcan redundancias y eviten inconsistencias. Las tres primeras formas normales (1FN, 2FN, 3FN) son las más utilizadas:

  • 1FN: no se permiten atributos repetidos ni campos multivaluados.
  • 2FN: cada campo depende completamente de la clave primaria.
  • 3FN: elimina dependencias transitivas entre columnas no clave.

Ejemplo de diseño normalizado

CREATE TABLE autores (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nombre VARCHAR(80) NOT NULL
);

CREATE TABLE libros (
  id INT AUTO_INCREMENT PRIMARY KEY,
  titulo VARCHAR(120) NOT NULL,
  anio INT,
  precio DECIMAL(10,2),
  id_autor INT,
  FOREIGN KEY (id_autor) REFERENCES autores(id)
);

De este modo, cada autor se almacena una sola vez y los libros referencian al autor mediante una clave foránea.

Proyecto guiado: Biblioteca

Diseña una base de datos llamada biblioteca con dos tablas relacionadas:

  • autores(id, nombre, pais)
  • libros(id, titulo, anio, precio, id_autor)

Inserta al menos tres autores y cinco libros. Realiza las siguientes consultas:

  1. Mostrar todos los libros ordenados por año descendente.
  2. Mostrar el precio medio de los libros por autor.
  3. Listar los autores con más de un libro publicado.

Actividades interactivas sugeridas

  • Rellenar huecos: completa las sentencias CREATE TABLE y ALTER TABLE.
  • Verdadero/Falso: “Una clave foránea garantiza la integridad referencial entre tablas”.
  • Emparejar: forma normal ↔ descripción.

Actividad final: Explica con tus propias palabras por qué normalizar una base de datos facilita las actualizaciones y evita errores de duplicidad.
